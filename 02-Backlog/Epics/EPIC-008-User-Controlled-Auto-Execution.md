# EPIC-008: User-Controlled Auto-Execution with Own Wallets

## Goal

Enable fully automated paper and live trade execution using the user's own exchange accounts (Hyperliquid for crypto perps, Alpaca for US equities). Signals generated by the HermesForge scanner are automatically placed as orders on the user's exchange, with stop-loss and take-profit orders managed programmatically. Performance is tracked in real-time and reported to Discord.

## Why This Is Deferred

Signals must demonstrate a validated edge before automating execution. Building execution infrastructure for unvalidated strategies is wasted effort and introduces real financial risk. This Epic is blocked until:

1. At least one strategy passes Phase 1C (paper trading validation) with ≥ 30 trades and positive expectancy
2. Phase 1A/1B scanner results are confirmed by real execution (fills, slippage, timing)
3. Risk framework v2.0 is defined for live trading (position sizing, drawdown limits, kill switch)

## Scope

### Hyperliquid (Crypto Perps)
- Official Python SDK: hyperliquid-python-sdk (`pip install hyperliquid-python`)
- Authentication: Ethereum agent wallet (EIP-712 signing) — dedicate a separate agent wallet, never expose main wallet private key
- Testnet: https://api.hyperliquid-testnet.xyz (free testnet USDC via faucet)
- Order types: entry limit/market + separate SL trigger order + TP trigger order (no native bracket — must manage leg cancellation in code)
- 24/7 operation — crypto strategies run on daily UTC bar close
- Intraday stop detection: native trigger orders monitor mark price continuously
- Known risk: no native OCO bracket — dangling order risk if connectivity lost after one leg fills; requires reconciliation check on startup

### Alpaca (US Equities)
- Simple API key + secret authentication
- Native bracket orders (entry + SL + TP as linked OCO group)
- Paper trading account: free at alpaca.markets
- Market hours only (9:30 AM–4:00 PM ET)
- Intraday stop detection: native stop orders monitor price continuously

## Dependencies
- EPIC-007 complete (strategy validation infrastructure)
- EPIC-009 complete (Discord signal distribution — signals must be validated and published before automating execution)
- Strategy B (or any strategy) passing Phase 1C with ≥ 30 paper trades and positive expectancy
- Risk framework v2.0 defined
